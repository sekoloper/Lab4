# Lab4
Программа представляет из себя первую часть лабораторной работы под номером 4 по дисциплине «Язык программирования C#».Первая часть лабораторной состоит из пяти задач.

## Требования
* Windows 10 (или выше)
* Visual Studio

## Установка и запуск программы
1. Клонируйте репозиторий в Visual Studio
2. Соберите решение (Ctrl+Shift+B)
3. Запустите (Ctrl+F5)

## Руководство по использованию
1. Введите номер задачи.
2. Следуйте дальнейшим инструкциям внутри задачи.
3. После выполнения задачи введите номер новой или 0 для выхода из программы.

## Структура проекта
Lab4
* Program.cs
* InputValidation.cs
* Tasks.cs
* TaskFunction.cs

Program.cs – основной файл, содержащий класс Program и метод Main. В нём в консоль выводится список задач и запрашивается номер задачи, после чего вызывается метод соответствующей задачи из класса Tasks.

InputValidation.cs – файл с функцией корректного ввода числа типа int

Tasks.cs – файл, в котором содержатся методы задач в классе Tasks.

TaskFunction.cs – файл, в котором содержится класс со статическими методами, решающими поставленные задачи

## Задачи

1. List

Формулировка: Составить программу, которая проверяет, есть ли в списке L хотя бы два одинаковых элемента.

Алгоритм: С помощью метода выполняет проход по списку двумя циклами, происходит сравнение элементов. Если два элемента равны, возвращается true, иначе false.

Результат:

Ваш список: 1 2 3 Есть ли в списке одинаковые элементы: False
Ваш список: 4 0 4 Есть ли в списке одинаковые элементы: True

2. LinkedList

Формулировка: Удалить из списка L первое вхождение заданного элемента, если такой есть.

Алгоритм: К списку применяется функция Remove, которая возвращает true при успешном удалении первого вхождении числа и false иначе. Результат удаления выводится.

Результат:
Ваш список: 3 2 2 Значение удаляемого элемента: 2 Первое вхождение элемента с указанным значением удалено. Ваш список: 3 2
Ваш список: 1 2 3 Значение удаляемого элемента: 4 Элемент не найден. Ваш список: 1 2 3

3. HashSet

Формулировка: Есть перечень музыкальных произведений. Определить для каждого произведения, какие из них
нравятся всем n меломанам, какие — некоторым из меломанов, и какие — никому из меломанов.

Алгоритм: создаются массив строк с названиями песен и словарь из строки (имя меломана) и hashset строкого типа, содержащего названия любимых песен.
В методе MusicPreferences создаётся список меломанов, которым нравится текущая песня. По массиву песен выполняется проход через цикл foreach.
Выполняется проход по словарю через цикл foreach. Если текущий hashset содержит название песни, имя текущего мелоамана заносится в список меломанов.
В соответствии с размером этого списка выводится соответсующая запись о том, нравится ли кому-то текущая песня.

Результат:

Песня Golden Brown нравится следующим меломанам: Комаров П.
Песня Soul Kitchen нравится всем меломанам.
Песня Долгая счастливая жизнь нравится следующим меломанам: Камаров Д.
Песня Мусорный ветер нравится следующим меломанам: Комаров П. Иванов И.
Песня Во время дождя нравится следующим меломанам: Камаров Д. Иванов И.
Песня Never More не нравится никому из меломанов.

4. HashSet 2

Формулировка: Файл содержит текст на русском языке. Напечатать в алфавитном порядке все гласные буквы,
которые не входят более чем в одно слово.

Алгоритм: Создаются hashset со всеми гласными русского языка и словарь из символа и числа (сколько раз этот символ встречается в разных словах).
В словарь заносятся все гласные, им присваивается значение 0.
Все слова из текста разделяются по специальным символам и заносятся в массив.
Выполняется проход по массиву. Создаётся hashset с гласными, входящими в это слово. Значение каждой гласной в словаре, входящей в текущий hashset, увеличивается на 1.
Создаётся результирующий hashset, в который переносятся гласные, чьё значение в словаре меньше 2 (то есть гласная входит в одно слово или не входит вообще).
Через цикл значения переностся из hashset в список (чтобы также провести верную сортировку), который метод возвращает.

Результат:

Все гласные буквы, которые не входят более чем в одно слово: е ё и о у ы э ю я

5. Dictionary/SortedList

Формулировка: На вход программе сначала подается число участников олимпиады N. В каждой из следующих N
строк находится результат одного из участников олимпиады в следующем формате:
<Фамилия><Имя><класс><баллы>
где <Фамилия>– символьная строка (не более 20 символов),<Имя>– символьная строка (не более
15 символов), <класс>– число от 7 до 11, <баллы>– целое число набранных участником баллов.
<Фамилия> и <Имя>, <Имя> и <класс>, а также <класс> и<баллы> разделены одним пробелом.
Пример входной строки:
Семенов Егор 11 225
Победителем олимпиады становится участник, набравший наибольшее количество баллов, при
условии, что он набрал более 200 баллов. Если такое количество баллов набрали несколько
участников, то все они признаются победителями при выполнении условия, что их доля не
превышает 20% от общего числа участников.
Победителем олимпиады не признается никто, если нет участников, набравших больше 200
баллов, или больше 20% от общего числа участников набрали одинаковый наибольший балл.
Напишите эффективную по времени работы и по используемой памяти программу, которая
будет определять фамилию и имя лучшего участника, не ставшего победителем олимпиады.
Если таких участников несколько, т.е. если следующий за баллом победителей один и тот же
балл набрали несколько человек, или, если победителей нет, а лучших участников несколько (в
этом случае именно они являются искомыми), то выдается только количество искомых
участников. Гарантируется, что искомые участники (участник) имеются.
Программа должна выводить через пробел фамилию и имя искомого участника или их
количество. Пример выходных данных (один искомый участник):
Семенов Егор
Второй вариант выходных данных (несколько искомых участников):
12

Алгоритм: 
Создаётся класс Participant с полями LastName, FirstName, Grade, Score.
Создаётся список участников типа Participant.
Из файла считываются данные об участниках и заносятся в список.
Создаётся SortedList, в котором по количеству баллов хранится список участников с соответствующим количеством баллов.
Создаётся баллов и находится максимальный балл.
Создаётся список участников с максимальным баллом (кандидатов в победители).
Если максимальный балл больше 200, процент кандидатов не больше 20 и есть люди, не ставшие победителями в список заносятся участники со следующим баллом после максимального.
Иначе кандидаты не являются победителями и список остаётся тем же.
Если в списке один человек, выводятся его фамилия и имя, иначе количество человек.

Также реализован ввод участников с проверкой входных данных через метод EnterResults.
